Come funziona

Quattro passaggi, completamente automatici. Tu non devi fare niente.

1

Rileva

Quando lo spazio scende sotto il 15%, ShadowFS trova i file grandi che non apri da almeno 3 giorni.

2

Trasferisce

Li invia al Raspberry su canale cifrato. Il file locale viene svuotato solo dopo la conferma di ricezione verificata.

3

Ghosta

Sul telefono resta una miniatura leggerissima. La galleria continua a mostrare l'anteprima, lo spazio è libero.

4

Restituisce

Apri il file (o premi Ripristina) e torna sul telefono in pochi secondi, integro e verificato. Dopo un'ora, torna ghost.

La sicurezza non è un'opzione

Ogni scelta di design parte da una domanda: "e se va storto?"

Mai perdita di dati

Il file sul telefono viene svuotato solo dopo che il Raspberry conferma: ricevuto, checksum verificato, salvato su disco. Nessuna conferma = file intatto.

mTLS bidirezionale

Non solo il telefono verifica il server: anche il Raspberry accetta solo telefoni con certificato firmato dalla TUA CA privata. Nessun altro entra.

TLS 1.3, niente legacy

Solo l'ultima versione del protocollo. Niente downgrade, niente cifrari deboli.

Trasferimenti atomici

File temporanei .part e .tmp: nessun download vede mai un file a metà, nessun ghost viene sostituito da un file corrotto.

Anti path-traversal

Ogni percorso è validato sul server: impossibile uscire dalla cartella di storage, anche con richieste malevole.

Pairing one-shot

Il QR di configurazione scade in 5 minuti e funziona una sola volta. Token casuale da 128 bit.

⚠️ E se uso già Google Photos o Amazon Photos?

Nessun problema — ShadowFS è progettato per conviverci. I backup fotografici cloud non vengono mai corrotti, e se un servizio "ripristina" un file ghostato, ShadowFS reagisce da solo:

  • Primo ripristino: re-ghost intelligente senza ricaricare nulla (il contenuto è già sul Raspberry)
  • Secondo ripristino: il file viene escluso dal ghosting e ricevi una notifica — il loop muore lì
  • Tetto di sicurezza: massimo 15 idratazioni automatiche all'ora, sempre

Domande frequenti

Le risposte oneste, senza marketing.

I miei file sono al sicuro? Davvero?

Sì, per design: il file sul telefono viene svuotato solo dopo che il Raspberry ha confermato di averlo ricevuto, verificato con checksum SHA-256 e salvato su disco. Se la rete cade, se il disco è pieno, se il checksum non torna — il file sul telefono resta intatto. Il Raspberry però è una copia singola: per i file insostituibili, un backup periodico del Raspberry stesso resta una buona idea.

Quanto spazio posso liberare?

Dipende da quanto pesa la tua libreria: tipicamente decine di GB tra video, foto e archivi. Vengono ghostati i file sopra i 512 KB non usati da 3 giorni — i file recenti e quelli piccoli restano sempre sul telefono.

E se apro una foto ghostata mentre sono al mare?

Con Tailscale attivo, il telefono raggiunge il Raspberry di casa anche in 4G: la foto si riscarica in pochi secondi. La miniatura intanto è sempre visibile in galleria.

Serve essere esperti per installarlo?

Serve saper copiare 3 comandi nel terminale del Raspberry — l'installer fa tutto il resto (Go, compilazione, certificati, servizio, firewall, QR). Lato telefono: scansioni il QR e concedi i permessi guidati. Dieci minuti in tutto.

Che differenza c'è con Google One / iCloud?

Stesso comfort, filosofia opposta: i file stanno su un hardware che possiedi, cifrati con certificati che generi tu, senza canone mensile e senza che nessuna azienda possa leggerli, analizzarli o chiuderti l'account.

Quali telefoni e Raspberry sono supportati?

Android 10 o superiore (consigliato 11+). Lato server qualsiasi Raspberry Pi — anche un vecchio Pi 3 con un disco USB — oppure qualsiasi server di casa collegato in rete: un vecchio PC, un NAS, un mini PC con Linux. Il daemon è scritto in Go ed è leggerissimo.

È open source?

Sì: daemon Go, app Android Kotlin e installer sono completamente ispezionabili. Il protocollo è documentato nel README — niente scatole nere per i tuoi dati.
