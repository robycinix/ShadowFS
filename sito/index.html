Flusso

Meno spazio occupato, accesso preservato.

ShadowFS lavora in background: individua i file usati di rado, li trasferisce, li verifica e lascia sul telefono un segnaposto leggero.

shadowfs scan --when-storage-low
> found 148 cold files
> upload over TLS 1.3 + mTLS
> verify sha256 checksums
> replace originals with verified placeholders
> restore from ghost list (auto-restore optional)

Sicurezza

Prima regola: non perdere mai un file.

La comodità ha valore solo se gli originali restano protetti. ShadowFS modifica il telefono solo dopo la conferma completa del server.

Verifica prima della sostituzione

Il file locale viene alleggerito solo dopo ricezione completa, SHA-256 verificato, salvataggio su disco e ACK finale.

  • Nessuna conferma, nessuna modifica locale.
  • File temporanei per upload e download incompleti.
  • Namespace isolato per ogni dispositivo.

TLS 1.3 + mTLS

Il telefono verifica il server e il server accetta solo telefoni con certificato firmato dalla tua CA privata.

Abbinamento monouso

Il QR scade in pochi minuti e vale una sola volta, con token casuale ad alta entropia.

Percorsi server protetti

I percorsi sono validati sul server per impedire richieste fuori dallo storage dedicato.

Cloud sync controllato

Limiti e rilevamento loop impediscono idratazioni automatiche infinite.

FAQ

Risposte chiare.

Come proteggete i miei file?

ShadowFS non svuota l'originale finché il server non conferma ricezione, checksum e salvataggio. Per i file insostituibili resta comunque consigliato un backup del server.

Quanto spazio posso liberare?

Dipende dalla libreria: video, foto RAW, PDF e archivi vecchi possono liberare decine di GB lasciando anteprime leggere sul telefono.

Funziona fuori casa?

Sì, usando una rete privata come Tailscale il telefono può raggiungere il server anche da 4G o da un altro Wi-Fi.

Servono competenze tecniche?

Serve familiarità minima con il terminale sul server. Lato telefono il pairing guidato via QR evita configurazioni manuali lunghe.

Devo mantenere un backup?

Sì. ShadowFS verifica i trasferimenti prima di modificare i file locali, ma non sostituisce una strategia di backup. Usalo prima con file non critici e mantieni una copia separata dei dati importanti.

Cosa succede se il server è spento?

ShadowFS non modifica i file locali se il server non è raggiungibile. I trasferimenti restano in attesa o riprendono quando la connessione torna disponibile.

Posso usare Google Photos o OneDrive?

Sì, ma è meglio evitare che due servizi gestiscano la stessa cartella. Puoi proteggere DCIM, WhatsApp o cartelle cloud e usare ShadowFS per download, video grandi, PDF e archivi.

Se elimino un file dal telefono, viene eliminato anche dal server?

No, non automaticamente. La copia sul server resta finché non decidi di pulire gli orfani dalla lista dei file gestiti.

I miei file passano dai vostri server?

No. ShadowFS comunica tra il tuo telefono e il tuo server domestico. Il progetto non richiede un account ShadowFS e non ospita i tuoi file.

Quanto costa usarlo?

ShadowFS non prevede abbonamenti. Usi il tuo hardware e il tuo disco. Restano eventuali costi personali, come corrente, storage, rete o servizi esterni che scegli di usare.
